# Build Week submission checklist

## Working project

- [ ] Project runs locally from `index.html`
- [ ] All acceptance tests completed
- [ ] No private production data
- [ ] Repository created
- [ ] Repository public with appropriate license, or private and shared with required judging addresses
- [ ] README includes setup instructions and sample-data explanation
- [ ] Codex build log complete
- [ ] Codex `/feedback` Session ID saved

## Devpost

- [ ] Project name
- [ ] Elevator pitch
- [ ] Project story
- [ ] Built-with tags
- [ ] Challenge category
- [ ] Code repository URL
- [ ] Try-it-out URL if available
- [ ] Image gallery
- [ ] Public YouTube demo link
- [ ] Additional questions answered
- [ ] Final preview checked

## Public demo video

- [ ] Under 3 minutes
- [ ] Public on YouTube
- [ ] Voiceover included
- [ ] Shows the project working
- [ ] Explains what was built
- [ ] Explains how Codex was used
- [ ] Explains how GPT-5.6 was used
- [ ] Contains no copyrighted music
- [ ] Contains no third-party trademarks without permission
- [ ] Contains no private EVA production data

## Suggested 3-minute structure

0:00–0:20 — The real problem
0:20–0:45 — Event dashboard and production overview
0:45–1:25 — Backline subtasks with PLAN → APPROVE → APPLY
1:25–1:55 — Safe “Mark complete” and audit trail
1:55–2:20 — Update dashboard and local reset
2:20–2:45 — How Codex and GPT-5.6 were used
2:45–3:00 — Impact and next steps
